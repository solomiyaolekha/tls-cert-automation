# TLS Certificate Automation Tool

Професійна CLI-утиліта для моніторингу статусу TLS/SSL сертифікатів, перевірки термінів їхньої дії та автоматизації процесу генерації запитів на підпис (CSR).

## 🚀 Основні можливості

- **Мережеве сканування:** Отримання даних про сертифікати безпосередньо через HTTPS-з'єднання.
- **Інтелектуальний моніторинг:** Автоматичне визначення статусів сертифікатів (Valid, Expiring Soon, Expired) на основі заданого порогу днів.
- **Пріоритезація:** Використання структури даних `Heap` (пріоритетна черга) для виведення найбільш критичних сертифікатів першими.
- **Автоматизація криптографії:** Генерація приватних ключів та CSR (Certificate Signing Request).
- **Локальне підписання:** Можливість створення самопідписаних сертифікатів для тестових середовищ.
- **Гнучкий інтерфейс:** Підтримка CLI-аргументів через `argparse` та інтерактивний режим із захистом від опечаток.

## 🛠 Технологічний стек

- **Python 3.13+**
- **Cryptography:** Основна бібліотека для роботи з ключами та сертифікатами.
- **Argparse:** Для обробки аргументів командного рядка.
- **Heapq:** Для алгоритмічного сортування критичних подій за датою завершення.

Ось інструкція, вибудувана у правильному логічному порядку, щоб запуск пройшов без помилок:

---

### 📦 **Встановлення**

1. **Клонуйте репозиторій:**
   ```bash
   git clone https://github.com/solomiyaolekha/tls-cert-automation.git
   ```

2. **Перейдіть у папку проєкту:**
   ```bash
   cd tls-cert-automation
   ```

3. **Встановіть необхідну бібліотеку:**
   ```bash
   pip install cryptography
   ```

---

### 🚀 **Використання**

Якщо ви хочете перевірити конкретні домени з власним порогом днів і автоматично підтвердити всі дії, використайте цю команду:

```bash
python main.py --domain google.com apple.com --expirationdays 60 --yes
```

**Довідка**
Ви завжди можете переглянути список доступних параметрів через стандартний прапорець допомоги:
```bash
python main.py --help
```
